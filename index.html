🆘 Help & Support

📘 Entry ৰ পৰা Audit চেক লৈ — সম্পূৰ্ণ পদ্ধতি
  1. ১) গোটৰ পৰিচয় পূৰণ কৰক — Home → 👤 পৰিচয় ত গোটৰ নাম, ID, পঞ্জিয়ন নং আদি সাঁচি থওক।
  2. ২) সদস্য যোগ কৰক — Home → 👥 গোটৰ সদস্য ত নাম, ফোন নং (WhatsApp বিলৰ বাবে বাধ্যতামূলক), মাহিলী কিস্তিৰ পৰিমাণ সাঁচি থওক।
  3. ৩) মাহিলী জমা/ঋণ Entry দিয়ক — Bottom nav ৰ জমা/ঋণ ত সদস্য বাছি পৰিমাণ দি "✓ সংৰক্ষিত হৈছে" টিপক। ইচ্ছা কৰিলে "📲 হোৱাটআপে বিল পঠিয়াওক" ত সদস্যক ৰিচিপ পঠাব পাৰে।
  4. ৪) কেচবহী Entry — Bottom nav ৰ কেচবহী ত গোটৰ বেংক/নগদ লেনদেন (চৰকাৰী অনুদান, খৰচ, বেংক সুদ আদি) সাঁচি থওক।
  5. ৫) Master Ledger চাওক — Bottom nav ৰ লেজাৰ ত সদস্য+কেচবহীৰ পৰা auto-calculate হোৱা গোটৰ সামগ্ৰিক হিচাপ চাব পাৰে।
  6. ৬) Layer-2 Financial Mapping চাওক — Master Ledger পেজৰ শেষত "📊 Layer-2 Financial Mapping" বুটামত Asset/Liability/Income/Expense/Equity হিচাপে সুসংগঠিত ডাটা পাব।
  7. ৭) Audit Report চেক কৰক — Settings/Audit পেজত সকলো হিচাপ মিলিছে নে নাই যাচাই কৰক।
  8. ৮) Data সুৰক্ষিত ৰাখক — Settings ত "💾 Data Backup কৰক" নিয়মিত টিপক; কোনো ভুলতে data হেৰালে "♻️ শেষ Backup ৰ পৰা Data Restore" ব্যৱহাৰ কৰক।
সদস্যৰ জমা/ঋণৰ data হঠাৎ ০ দেখুৱাইছে, কি কৰিম?

Settings ত "♻️ শেষ Backup ৰ পৰা Data Restore কৰক" বুটাম টিপক। যদি নিয়মিত Backup লৈ থাকে, তাৰ পৰা data ঘূৰাই পাব। এই কাৰণেই নিয়মিত "💾 Data Backup" কৰাটো অতি প্ৰয়োজনীয়।

"হোৱাটআপে বিল পঠিয়াওক" টিপিলে একো নহয় কিয়?

সেই সদস্যৰ ফোন নং Group সদস্য tab ত save করা নাই। 👥 গোটৰ সদস্য ত গৈ সদস্যজনৰ 10-digit ফোন নং যোগ কৰি পুনৰ চেষ্টা কৰক।

Master Ledger আৰু Layer-2 Financial Mapping ৰ পাৰ্থক্য কি?

Master Ledger এ ২৩ টা পৃথক হিচাপ (সঞ্চয়, সুদ, খৰচ আদি) পৃথকভাৱে দেখুৱায়। Layer-2 Mapping এ সেই একেটা ডাটাকেই Asset/Liability/Income/Expense/Equity — এই ৫ টা standard accounting category ত সুসংগঠিত কৰি audit-উপযোগীকৈ দেখুৱায়। দুয়োটাৰ উৎস একেটাই, তেওঁলোক সদায় sync থাকে।

Settings ৰ "🧹 Reset" বুটামবোৰ কি কৰে?

এই বুটামবোৰ বৰ্তমান Production build-ত সুৰক্ষাৰ বাবে বন্ধ কৰা আছে (accidental data loss ৰোধ কৰিবলৈ)। প্ৰয়োজন হ'লে আগতে Backup লওক।

Admin PIN/Password পাহৰি গ'লে কি কৰিম?

প্ৰথমবাৰ যিটো PIN/Password দিয়া হৈছিল, সেয়া localStorage ত hash হিচাপে save থাকে — সেয়ে সঠিক PIN নজনালে পুনৰ সলনি কৰিব নোৱাৰি। Data সুৰক্ষিত ৰাখিবলৈ কোনো "backdoor" ইচ্ছাকৃতভাৱে ৰখা হোৱা নাই। যদি সঁচাকৈ পাহৰি গৈছে, Feedback tab ৰ পৰা সহায় বিচাৰক।

এটা সদস্যৰ পুৰণি (আগৰ মাহৰ) বিল পুনৰ চাব পাৰিমনে?

হয়, সদস্যৰ profile ত monthlyHistory অনুসৰি প্ৰতি মাহৰ snapshot সাঁচি থোৱা হয় — সদস্য বাছি পুৰণি মাহ চাব পাৰে।
